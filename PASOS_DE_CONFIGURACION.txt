Hola, si quieres saber exactamente cada click y comando que hicimos

========================================
 PASO 1: ENVIAR EL CODIGO A GITHUB
========================================
1. En mi computadora (local), tenia la carpeta de "Interstellar-main".
2. Abrimos la terminal (PowerShell o CMD) dentro de esa carpeta.
3. Usamos los comandos de Git para enlazar esta carpeta a un nuevo 
   repositorio en mi cuenta de GitHub:
   -> git add .
   -> git commit -m "Subiendo Ghosty inicial"
   -> git push
4. Ahora, en vez de estar solo en mi PC, todo el codigo fuente 
   estaba seguro publicamente/privadamente en la nube de GitHub.


========================================
 PASO 2: CREAR EL SERVIDOR EN KOYEB
========================================
1. Entramos a la pagina web de Koyeb (koyeb.com) y nos creamos/iniciamos sesion.
2. Le dimos click en "Create Web Service" (Crear Servicio Web).
3. Seleccionamos "GitHub" como metodo de despliegue.
4. Koyeb se conecto a mi cuenta y me mostro la lista de mis repositorios. 
   Escogimos el de "Ghosty" que subi en el Paso 1.
5. Dejamos todas las configuraciones casi por defecto (Koyeb detecta que 
   es un archivo NodeJS/npm y sabe como arrancarlo). 
6. Le dimos a "Deploy" (Desplegar). En un par de minutos, Koyeb instalo todo,
   corrio el servidor Ultraviolet y nos entrego un link verde oficial 
   ejemplo: (ghosty-marlon.koyeb.app).


========================================
 PASO 3: ESQUIVAR EL FILTRO CON CLOUDFLARE
========================================
*Nota: A veces las escuelas bloquean urls con la palabra "koyeb". 
 Aqui hicimos la magia negra para evitar eso.*

1. Entramos a Cloudflare.com y creamos una cuenta.
2. En el panel lateral, fuimos a la seccion "Workers & Pages".
3. Le dimos a "Create Application" -> "Create Worker" (Crear un Worker).
4. Le pusimos un nombre cualquiera y le dimos "Deploy" para crearlo.
5. Luego le dimos a "Edit Code" (Editar Codigo). Borramos todo lo que 
   traia por defecto y pegamos nuestro propio "Script de Proxy" (ese script
   que toma TODAS las peticiones y las manda a tu link de Koyeb de forma oculta).
6. En ese codigo reemplazamos la "URL de Target" por el link que nos habia 
   dado Koyeb en el Paso 2.
7. Click en "Save and Deploy" (Guardar y desplegar).
8. Cloudflare nos regalo un link nuevo inofensivo que termina en ".workers.dev".
   ¡Este link nuevo es el que le das a las demas personas y la escuela no podra!


========================================
 PASO 4: REDISEÑO 3D Y ACTUALIZACIONES
========================================
1. A partir de ahi, cada vez que quisimos cambiar colores, textos, limpiar tags

   computadora (los archivos global.css, settings.html, h.css, m1.js, etc).
2. Cuando el diseño estaba listo y "pulido", nada mas abrimos la terminal y:
   -> git add . ; git commit -m "Reescribiendo UX 3D" ; git push
3. Al hacer eso, GitHub se dio cuenta del cambio y automaticamente le 
   aviso a Koyeb.
4. Koyeb descargo el nuevo codigo, reinicio los servidores en segundo plano 
   sin que nos dieramos cuenta, y en 3 minutos nuestros usuarios ya tenian 
   la nueva interfaz con los botones "Stealth Cloak" y "Proxy Engine Core"
